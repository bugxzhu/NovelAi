### 1a: Add DiscussError to errors.py

Append to `app/memory/errors.py`:

```python
class DiscussError(Exception):
    """LLM discuss failed (invalid JSON, max_tokens, etc.)."""
```

### 1b: Create app/models/discuss.py

```python
from pydantic import BaseModel


class DiscussBranch(BaseModel):
    label: str
    title: str
    summary: str
    conflicts: str
    opportunities: str
    character_impact: str


class DiscussRequest(BaseModel):
    question: str


class DiscussResponse(BaseModel):
    question: str
    branches: list[DiscussBranch]
    recommended: str
    reasoning: str
    log_id: int
```

### 1c: Create prompts

`app/llm/prompts/discuss/system.j2` — content from spec §4.1.

`app/llm/prompts/discuss/user.j2` — same structure as `reviewer/user.j2` (project/world_overview/chapter/characters/relationships/events/lore/milestones/summaries) + question at end. **IMPORTANT:** copy reviewer/user.j2 structure including all context sections (characters, character_states_history, relationships, events, lore, plot_lines, milestones, recent_chapter_summaries). 请从 3 个不同方向推演这个设想的后果，并给出推荐。

## Task 2: Discuss agent + API endpoint

### 2a: Create app/agents/discuss.py

Follow `app/agents/reviewer.py` exactly. Key differences:
- `assemble_review_context` (reuse)
- Render `discuss/system.j2` + `discuss/user.j2` (pass `question` to user.j2)
- Parse JSON: `branches` (list), `recommended` (str), `reasoning` (str)
- Tolerance: branches not list → DiscussError; >3 → truncate to 3; <3 → keep as-is; missing recommended → "A"; missing reasoning → ""
- generation_logs: `beat_text="(discuss)"`, `instruction=question`, `model_task="discuss"`

### 2b: Create app/api/chapters_discuss.py

Follow `app/api/chapters_review.py` exactly. Endpoint: `POST /{chapter_id}/discuss` accepting `DiscussRequest` body.

Register in main.py.
